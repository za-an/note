版本：v0.1（Demo 阶段）
日期：2026-07-12
状态：待评审

## 一、Demo 定位与范围

**目标**：产出一个可演示的横屏鸿蒙应用 Demo，跑通「录音 → 云端转写 → RAG 提纲/问答 → 班级共享」核心闭环，验证产品价值主线。

**本期不做**（明确排除，但架构预留）：
- UI 动画（转场、隔空抓取动效）——组件结构与状态管理预留动画挂载点
- NFC 配对、跨设备流转、局域网互抓
- 端侧 ASR / 端侧 RAG
- 错题归集、知识点热力图
- 正式的账号注册体系（Demo 用预置账号）

## 二、核心概念与数据模型

### 2.1 核心概念

| 概念 | 说明 |
|------|------|
| **课程集合（Class-Course）** | 系统的基本组织单位 = 一个班级的一门课程（如「计科2301班-数据结构」）。**每个集合拥有独立的 RAG 知识库，集合之间知识不共享、检索不互通。** |
| 课时（Session） | 集合下的一次课，对应一次录音与一份转写文本、一份提纲 |
| 知识库（KB） | 归属于唯一集合，由该集合内已发布提纲与已共享的优质笔记构成 |
| 角色 | 教师（创建集合、审核发布）、学生（加入集合、记笔记、问答） |

### 2.2 数据实体关系

```
User (师/生)
  └─ 加入多个 → ClassCourse（课程集合）
                  ├─ Session（课时）1:N
                  │    ├─ Recording（音频文件）
                  │    ├─ Transcript（转写文本，带时间戳分段）
                  │    └─ Outline（Markdown 提纲：草稿/已发布）
                  ├─ Note（笔记）1:N —— 隐私 / 共享 两种可见性
                  ├─ KnowledgeBase（向量库，集合唯一，物理隔离）
                  └─ ChatSession（AI 问答会话）1:N
```

**知识库隔离规则**：向量库按 `class_course_id` 分库（每个集合独立 collection），检索请求必须携带集合 ID，服务端强制过滤，任何跨集合检索直接拒绝。

## 三、界面需求

### 3.1 全局规范

- **横屏优先**：应用锁定横屏（landscape），适配平板与折叠屏展开态；布局采用「左侧导航/列表 + 右侧内容区」双栏（局部三栏）结构，利用横向空间
- **简洁风**：大量留白、无衬线字体、单主色（建议蓝/青色系）+ 中性灰阶、卡片圆角、无渐变无重投影；图标用系统 HarmonyOS Symbol
- **动画预留**：所有页面切换走统一 Navigation 容器、卡片组件独立封装并暴露状态属性，后续可挂属性动画与共享元素转场，不改结构

### 3.2 页面清单

#### P1 首页 / 集合列表
- 左栏：用户信息、「我的课程集合」列表（卡片：课程名、班级、教师、课时数）
- 右栏：选中集合的概览（最近课时、最新发布提纲、快捷入口：开始录音 / AI 问答 / 我的笔记）
- 教师可「创建集合」，学生凭邀请码「加入集合」

#### P2 课堂录音页
- 顶部：课程集合名 + 本次课时标题（可编辑）
- 中部：录音状态（时长、音量波形占位图——静态即可）、实时转写文字流（云端 ASR 返回后滚动追加）
- 操作：开始 / 暂停 / 结束；结束后进入转写整理态
- 弱网处理：录音本地保存，转写失败可整段重传

#### P3 课时详情页（三栏）
- 左栏：该集合的课时列表
- 中栏：转写文本（按时间戳分段，可滚动）
- 右栏：AI 生成的 Markdown 提纲预览
- 操作：「生成提纲」（调 AI 层）；教师额外有「发布到共享池」（发布后写入该集合知识库）

#### P4 AI 问答页（Obsidian Copilot 式：编辑器为主体 + AI 侧栏）

交互范式参考 Obsidian 接入 AI 的形式（Copilot / Text Generator 插件）：**笔记编辑器是主体，AI 是可唤出的侧边栏面板与编辑器内命令**，而非独立聊天应用。

- **主区：MD 笔记编辑器**——编辑本集合下的一篇笔记（新建或选择已有）
- **右侧 AI 面板（可收起）**：
  - 对话区，流式输出；每条回答标注引用来源（提纲/笔记片段）
  - 回答操作：「插入到光标处」「替换选中内容」「复制」
  - 上下文控制：可勾选「携带当前笔记作为上下文」；输入 `@` 可引用本集合内其他笔记/提纲片段加入上下文（Obsidian 的 @ 引用形式）
- **编辑器内 AI 命令**（选中文本后浮出菜单，Obsidian 划词命令形式）：
  - 「问 AI」——将选中文本作为问题发到侧栏
  - 「总结」「解释」「续写」——结果以插入/替换方式回填编辑器
- 左栏（可折叠）：历史会话列表 + 本集合笔记列表
- **约束提示常驻**：「AI 仅依据本课程知识库作答」；知识库无相关内容时明确回复"本课程资料中未找到"，不得编造（编辑器命令如「续写」「解释」同样以集合知识库为检索范围）

#### P5 笔记页
- 左栏：我的笔记列表（含类型标识：📝 MD 笔记 / ✍️ 手写笔记；可见性标识：🔒私有 / 👥共享）
- 右栏：按笔记类型呈现对应编辑器，**两类笔记独立成篇，创建时二选一**：
  - **MD 笔记**：Markdown 编辑 + 实时预览（Demo 从简：标题/加粗/列表/图片/代码块）
  - **手写笔记**：全屏画板，支持**华为压感笔（M-Pencil）**——压感调节笔迹粗细，支持笔/橡皮/套索基础工具、多页管理；笔迹以矢量数据存储
- 操作：切换隐私/共享；共享后进入 AI 质量评估队列，评估通过 → 纳入该集合知识库（本期仅 MD 笔记参与评估与向量化，手写笔记可共享浏览但不入库，后期接手写识别后再入库）
- 导出：MD 笔记导出 Markdown；手写笔记导出图片/PDF（本期可选）

#### P6 教师审核页（仅教师角色）
- 待审核列表：AI 生成提纲（草稿）
- 右侧预览 + 「编辑」「发布」「退回」操作

### 3.3 页面流转

```
P1 首页 ──选集合──▶ P3 课时详情 ◀── P2 录音结束后跳转
   │                    │
   ├──▶ P2 录音          ├──▶ P4 问答（携带集合ID）
   ├──▶ P4 问答          └──▶ P6 审核（教师）
   └──▶ P5 笔记
```

## 四、功能需求

### F1 录音与云端转写
- F1.1 端侧 AudioCapturer 录音，保存为 wav/pcm 分片（60s/片）
- F1.2 分片实时上传服务端 → 服务端调用**云端 ASR**（华为云 SIS 或讯飞，二选一，接口做适配层可替换）→ 转写结果按分片回推客户端（WebSocket/SSE）
- F1.3 转写文本带时间戳，落库到对应 Session
- F1.4 失败重试：分片上传失败自动重试 3 次，仍失败标记待补传

### F2 提纲生成（AI 层）
- F2.1 转写完成后可手动触发「生成提纲」
- F2.2 服务端 RAG 管线：文本清洗 → 分块 → LLM 生成结构化 Markdown 提纲（章节 + 知识点 + 例题标注）
- F2.3 提纲状态机：`草稿 → 待审核 → 已发布 / 已退回`；仅「已发布」的提纲向量化写入该集合知识库

### F3 集合限定 AI 问答
- F3.1 问答请求必须绑定 class_course_id；检索仅命中该集合知识库
- F3.2 检索 Top-K 片段 + 问题拼 prompt → LLM 流式回答；回答附引用来源
- F3.3 无命中时按固定话术回复，不做通用回答（防止脱离课堂内容）

### F4 笔记与共创资源池
- F4.1 笔记 CRUD，默认私有；笔记分两种独立类型：**MD 笔记**（Markdown 文本）与**手写笔记**（矢量笔迹，多页）
- F4.2 切换共享 → LLM 质量评估（相关性、正确性、结构性三维打分，阈值可配）→ 通过后向量化入该集合知识库，列表展示「已入库」徽标（本期仅 MD 笔记参与评估入库；手写笔记可共享浏览，不入库）
- F4.3 导出：MD 笔记导出 Markdown 到本地；手写笔记导出图片/PDF（本期可选）
- F4.4 AI 融合编辑（Obsidian 式）：AI 侧栏回答支持插入光标处/替换选区；编辑器划词唤起「问 AI / 总结 / 解释 / 续写」命令；`@` 引用集合内笔记与提纲作为对话上下文
- F4.5 手写输入：接入压感笔事件（压力值映射笔迹粗细），笔迹数据结构预留倾角/速度字段，便于后期升级笔锋效果与手写识别
- F4.6 命令与上下文机制做成可注册结构（命令注册表 + 上下文提供者接口），后期新增 AI 命令（如「生成例题」「制卡复习」）无需改动编辑器核心

### F5 集合与角色管理
- F5.1 教师创建集合，生成 6 位邀请码；学生输码加入
- F5.2 Demo 预置账号：1 教师 + 2 学生，免注册直接选择身份登录
- F5.3 权限：审核/发布仅教师；录音师生均可（学生录音生成的提纲同样进入教师审核流）

## 五、技术方案（Demo 版）

| 模块 | 选型 | 说明 |
|------|------|------|
| 客户端 | HarmonyOS NEXT + ArkTS/ArkUI | 横屏锁定；Navigation 路由；@ComponentV2 状态管理 |
| 录音 | AudioCapturer + 长时任务 | pcm 分片 |
| 通信 | REST（业务）+ WebSocket（转写回推）+ SSE（问答流式） | |
| 服务端 | Python FastAPI | 业务 + AI 同栈，Demo 单体部署 |
| ASR | **云端，服务商待定**：适配层接口 `ASRProvider`（提交分片 / 拉取结果 / 回调），先内置 Mock 实现（返回预置转写文本）保证联调不阻塞，华为云 SIS / 讯飞后续任选其一实现接入 | |
| RAG | LangChain + bge-m3 embedding + **Chroma（按集合分 collection）** | 集合级物理隔离 |
| LLM | **OpenAI 兼容接口**（`base_url` + `api_key` + `model` 三项可配），可对接 DeepSeek / 通义 / vLLM 自部署等任意兼容服务 | 提纲生成 / 问答 / 笔记评估共用同一客户端 |
| 手写 | ArkUI Canvas + 手写笔事件（pointer 压力值）；笔迹存自定义矢量 JSON | 模拟器阶段以鼠标模拟（压力恒定），真机再验证压感 |
| 数据库 | 服务端 PostgreSQL；端侧 RelationalStore 缓存 | |
| 文件存储 | Demo 用服务器本地磁盘（MinIO 预留） | |

## 六、非功能需求

- **性能**：转写回显延迟 ≤ 5s/分片；问答首 token ≤ 3s；提纲生成 ≤ 60s（45 分钟课时）
- **隔离性**：跨集合检索 0 泄漏（服务端强制过滤 + 分 collection 双保险）
- **可扩展**：ASR / LLM 均走适配层接口；UI 组件预留动画属性；协同层（NFC/流转）以独立模块目录占位
- **Demo 数据**：内置 1 个示例集合 + 1 段示例课程音频/转写文本，保证无网络录音条件下也能演示后续流程

## 八、演示环境与已确认事项

- **演示环境**：初期以 DevEco Studio 模拟器横屏演示为准；压感笔效果在模拟器中以鼠标代替（压力恒定），压感差异需真机（MatePad + M-Pencil）验证，作为后续事项
- ASR：走适配层，服务商暂不锁定；Demo 默认启用 Mock 实现，接入真实服务商时仅新增一个 Provider 实现
- LLM：OpenAI 兼容协议，具体服务通过配置切换
- 学生允许录音，其提纲进入教师审核流
- 笔记质量评估阈值 Demo 阶段写死

- 手写笔记独立成篇（不与 MD 混排）；本期不参与知识库入库，后期接手写识别后再入库
