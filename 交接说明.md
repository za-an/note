## 二、协同层实现（全部需真机验证）

### 1. NFC 碰一碰（`service/collab/NfcService.ets` + 协同页）
- 方案：NFC 标签（NDEF 文本）承载课程邀请码——教师「写卡」贴教室，学生「碰一碰」读码自动入班
- 说明：HarmonyOS NEXT 三方应用无设备间 NFC P2P，标签中介是可落地的碰一碰形态
- API：`tag.registerForegroundDispatch` 前台分发 + `NdefTag.readNdef/writeNdef`
- 权限：`ohos.permission.NFC_TAG`（已声明）

### 2. 局域网互抓（`service/collab/TransferService.ets` + 协同页）
- 方案：UDP 广播发现（端口 41520）+ TCP JSON 协议传笔记（端口 41521），公开 socket API 实现
- 流程：A 开「共享我的笔记」开关 → B「发现设备」→ 点开设备 → 逐篇「抓取」→ 自动存为 B 的笔记（标题带来源）
- 限制：模拟器 NAT 不支持广播，需两台真机同一 WLAN

### 3. 跨端接续（`EntryAbility.ets` + `GlobalState` 快照）
- `module.json5` 已设 `"continuable": true`；切页签/进课程时实时打包快照（用户+课程+页签+课时）
- 源端 `onContinue` 写入 wantParam；目标端 CONTINUATION 启动时恢复状态直达原页面
- 权限：`ohos.permission.DISTRIBUTED_DATASYNC`（协同页有「开启流转授权」按钮做运行时申请）
- 触发方式：同华为账号 + 同 WLAN，另一设备任务中心/接续入口选本应用

## 四、已知风险点（无法本地编译验证，报错优先查这些）

| 位置 | 风险 | 快速修法 |
|------|------|----------|
| NfcService | `tag.registerForegroundDispatch` 回调签名 / `tag.ndef.makeTextRecord` 命名空间在不同 SDK 版本有差异 | 对照 DevEco 里 `@ohos.nfc.tag` d.ts 微调 |
| TransferService | `socket.UDPSendOptions/TCPConnectOptions` 字段、UDP 广播是否需 `setExtraOptions` | 对照 `@ohos.net.socket` d.ts；广播失败时先用固定 IP 直连测 TCP |
| EntryAbility | `onContinue` 同步/异步签名（部分版本支持 Promise） | 保持同步返回 AGREE 即可 |
| Index | `@StorageLink+@Watch` 组合恢复逻辑 | 若不触发，改用 AppStorage 轮询或 onPageShow |
| CollabPage | emoji 图标在部分模拟器字体缺字 | 换成文字或 SymbolGlyph |
