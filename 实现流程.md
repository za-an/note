# 智慧伴学系统 · 实现流程

> 依据《介绍文档.md》整理：模块划分、各模块技术栈、推荐实现顺序与里程碑。

---

## 一、总体架构

```
┌────────────────────────────────────────────────┐
│                 鸿蒙客户端 (ArkTS)               │
│  UI层(ArkUI) │ 笔记编辑 │ AI问答 │ 协同流转      │
├────────────────────────────────────────────────┤
│ 采集层        │ AI层      │ 业务层   │ 协同层    │
│ 录音/转写     │ RAG知识库 │ 审核/评估 │ NFC/流转 │
├────────────────────────────────────────────────┤
│        服务端（班级共享池 / 账号 / 审核后台）      │
└────────────────────────────────────────────────┘
```

## 二、模块划分与技术栈

### 1. 客户端基础框架（App 骨架）
- **技术栈**：HarmonyOS NEXT + ArkTS + ArkUI（声明式 UI）、Stage 模型、DevEco Studio
- **内容**：页面路由（Navigation）、状态管理（@State / AppStorage / V2 状态装饰器）、主题与动画（属性动画、转场动画）、本地存储（Preferences + RelationalStore/SQLite）

### 2. 采集层：录音与语音转写
- **技术栈**：
  - 录音：`@ohos.multimedia.audio`（AudioCapturer）或 AVRecorder
  - 转写：HarmonyOS AI 语音识别 Core Speech Kit（端侧实时转写）；长录音可备选云端 ASR（讯飞/华为云 SIS）
- **内容**：后台长时录音（长时任务 ContinuousTask）、实时转写字幕、转写文本分段落存储、时间戳对齐

### 3. AI 层：RAG 知识库 + 提纲生成
- **技术栈**：
  - 文本切分/向量化：服务端 Python（LangChain / LlamaIndex）+ Embedding 模型（bge-m3 等）
  - 向量库：轻量场景 SQLite-vec / Chroma；规模化用 Milvus
  - 生成模型：大模型 API（如 DeepSeek / 华为盘古 / 其他 LLM API）生成 Markdown 提纲与问答
  - 端侧可选：HarmonyOS 端侧 AI（HiAI / 端侧小模型）做离线摘要，作为差异化亮点
- **内容**：转写文本 → 清洗分块 → 知识点抽取 → 向量入库 → 提纲生成（Markdown）→ 课程限定问答（检索增强，仅依托本课知识库作答）

### 4. 业务层：共享池与内容管理（服务端）
- **技术栈**：
  - 后端：Node.js (NestJS) 或 Python (FastAPI)，与 AI 层同栈可选 FastAPI
  - 数据库：PostgreSQL / MySQL；对象存储存音频与笔记附件（华为云 OBS 或 MinIO）
  - 接口：RESTful + WebSocket（问答流式输出用 SSE）
- **内容**：账号与班级体系（师/生角色）、教师审核发布流程、笔记隐私/共享模式、AI 内容质量评估（LLM 打分）→ 优质笔记反哺知识库、知识点热力图与错题归集统计

### 5. 笔记模块（客户端）
- **技术栈**：ArkUI RichEditor（图文编辑）、Markdown 渲染（自研解析或三方库）、导出用服务端渲染 PDF（如 Puppeteer/weasyprint）或端侧生成
- **内容**：图文笔记编辑、Markdown 提纲查看、MD/PDF 导出、隐私/共享切换

### 6. 鸿蒙协同层（差异化核心）
- **技术栈**：
  - NFC 配对：`@ohos.nfc` (Connected Tag / NFC Kit)
  - 局域网文件互抓（“隔空抓取”）：分布式软总线 / Share Kit、DeviceManager 设备发现
  - 跨端接续：`continuation` / Ability 跨端迁移（流转 Continuation Kit）、分布式数据对象同步学习进度
- **内容**：NFC 碰一碰建立配对 → 局域网拖拽抓取文件（含抓取动画 UI）→ 平板/手机/PC 跨设备接续学习进度

## 三、实现顺序（5 个阶段）

### 阶段 1：地基（第 1–2 周）
1. 搭建 ArkTS 工程骨架：路由、主题、核心页面框架（首页/课程/笔记/我的）
2. 服务端脚手架：账号、班级、课程的基础 CRUD 与数据库设计
> 里程碑：能登录、建班级、看到空课程列表

### 阶段 2：采集层（第 3–4 周）
1. 录音功能（含后台长时录音）
2. 实时语音转写 + 转写文本存储、时间戳对齐
> 里程碑：一节课录音 → 得到完整转写文本

### 阶段 3：AI 层（第 5–7 周，核心闭环）
1. 服务端 RAG 管线：分块 → 向量化 → 入库
2. Markdown 提纲自动生成
3. 课程限定 AI 问答（检索 + 流式回复）
> 里程碑：转写文本 → 提纲 + 可问答，「学-复习」闭环打通

### 阶段 4：业务层（第 8–9 周）
1. 教师审核发布提纲到班级共享池
2. 图文笔记编辑、隐私/共享模式
3. AI 质量评估 → 优质笔记扩充知识库
4. MD/PDF 导出、知识点热力图、错题归集
> 里程碑：师生共创资源池可用

### 阶段 5：鸿蒙协同层 + 打磨（第 10–12 周）
1. NFC 碰一碰配对
2. 局域网文件互抓 + 抓取动画
3. 跨设备学习接续
4. 整体 UI 动画打磨、多设备联调、试点测试
> 里程碑：完整 Demo，可参赛/试点

## 四、顺序设计的理由

- **先闭环后亮点**：录音→转写→提纲→问答是产品价值主线，优先打通；NFC/流转是差异化亮点但依赖多设备调试，放后期集中攻克。
- **AI 层放服务端起步**：端侧 RAG 调试成本高，先用服务端 LangChain 快速验证效果，后期再评估端侧迁移作为「端侧 RAG」卖点。
- **协同层并行预研**：NFC 与分布式流转 API 权限、真机要求较多，建议阶段 2 起由一人并行做技术验证 Demo，阶段 5 集成。

## 五、风险提示

| 风险 | 应对 |
|------|------|
| 端侧语音转写长时稳定性 | 备选云端 ASR，分段上传 |
| 跨端流转需同华为账号+真机 | 提前准备 2+ 台鸿蒙真机联调 |
| LLM API 成本与延迟 | 缓存提纲结果，问答用流式输出 |
| 教师审核参与度 | 提供一键采纳 AI 提纲的低成本操作 |
